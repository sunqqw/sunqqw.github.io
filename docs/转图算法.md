# 拼豆转图算法：描边优先的色号量化管线

本文介绍拼豆图纸生成器中将任意图片转换为拼豆网格的核心算法。整套管线在浏览器端通过 Web Worker 运行，不依赖服务端。

---

## 1. 问题定义

**输入：**

- 一张 RGBA 图片（JPG / PNG）
- 目标网格尺寸 \(W \times H\)（如 50×50）
- 拼豆色号库（如 MARD 291 色）
- 用户参数：精细度、背景容差、相似度阈值、最小区域

**输出：**

- 一个 \(W \times H\) 的网格，每格对应一个色号 ID 或空白（`null`）

**核心难点：**

1. **抗锯齿**：插画边缘存在大量灰度中间色，直接降采样会产生色号库中不存在的「虚拟灰」
2. **细描边丢失**：黑色轮廓在源图中可能只有 1–2 像素宽，块均值或简单多数投票会将其淹没
3. **轮廓不完整**：降采样后，贴靠背景的填充色格缺少描边，出现「粉色直接挨着白色背景」
4. **内部镂空**：眼睛、高光等内部空白不能被误补为描边

## 3. 颜色角色分类

在后续步骤中，需要将色号分为三类，而非一视同仁地处理。

```typescript
type ColorRole = 'background' | 'outline' | 'fill'
```

分类规则（基于 RGB 亮度与饱和度）：

| 角色 | 判定条件 | 典型例子 |
|------|----------|----------|
| `background` | 亮度 > 230 且饱和度 < 0.18 | 白色 H1 |
| `outline` | 亮度 < 75，或亮度 < 110 且饱和度 < 0.25 | 黑色 H7、深灰 |
| `fill` | 其余 | 粉色 F21、黄色 A17 |

实现见 `lib/image/color-role.ts`。

**设计意图：** 描边和填充在拼豆图纸中有不同的结构作用。描边定义外形，填充表达体积。后续所有合并、清理操作都只对 `fill` 生效，从架构上保护描边不被吞掉。

---

## 4. 背景移除：边界洪水填充

### 算法

1. 将图片四边所有像素加入 BFS 队列
2. 从队列取出像素 `p`，标记为背景
3. 检查 `p` 的四邻域像素 `q`：若 `q` 与 `p` 的 RGB 距离 ≤ 容差，则将 `q` 入队
4. 重复直到队列为空
5. 所有被标记为背景的像素，Alpha 置为 0

### 容差换算

用户参数 `bgTolerance`（0–100）通过 `toleranceToDistanceSq()` 映射为 RGB 欧氏距离的平方阈值：

\[
d_{\max} = \frac{\text{tolerance}}{100} \times 441.67
\]

### 特点

- 只移除与**图片外边界连通**的相似色区域
- 角色内部的白色（如眼睛高光）因不与外边界连通，不会被移除
- 不做额外的「抗锯齿剥离」，避免侵蚀主体边缘

---

## 5. 全图色号量化

### 算法

对源图中每个 Alpha ≥ 128 的像素，用 RGB 欧氏距离在色号库中找最近色：

```typescript
colorIds[i] = matcher.findNearest(r, g, b)
```

透明像素标记为空字符串 `''`。

### 为什么先量化再降采样

| 方案 | 问题 |
|------|------|
| 先降采样再匹配 | 块内灰度均值落在两个色号之间，产生毛边 |
| 先量化再降采样 | 每个源像素已是离散色号，块内只统计真实色号 |

这一步将连续的抗锯齿过渡「坍缩」为色号库中的离散颜色，是消除灰色毛边的基础。

`PaletteMatcher` 内部对 `packRgb(r,g,b)` 做缓存，避免重复计算。

## 6. 描边优先降采样

将源图 \(S_w \times S_h\) 映射到目标网格 \(W \times H\)。

### 块划分

每个网格格 \((g_x, g_y)\) 对应源图中的矩形区域：

\[
\begin{aligned}
x_0 &= \lfloor g_x \cdot S_w / W \rfloor \\
x_1 &= \min(\lceil (g_x+1) \cdot S_w / W \rceil,\; S_w) \\
y_0 &= \lfloor g_y \cdot S_h / H \rfloor \\
y_1 &= \min(\lceil (g_y+1) \cdot S_h / H \rceil,\; S_h)
\end{aligned}
\]

### 分桶投票

遍历块内所有非透明像素，按颜色角色分入两个桶：

- `outline` 桶：描边色色号及票数
- `fill` 桶：填充色色号及票数

（背景色票不计入。）

### 决策逻辑

设块内不透明像素总数为 \(N\)，描边票总数为 \(O\)，描边保留阈值：

\[
T = \max(1,\; N \times r_{\text{outline}})
\]

其中 \(r_{\text{outline}} = 0.14 - \frac{\text{detail}}{100} \times 0.1\)。

| 条件 | 输出 |
|------|------|
| \(O \geq T\) 且有描边色 | 票最多的描边色 |
| 否则，有填充色 | 票最多的填充色 |
| 否则，有描边色 | 票最多的描边色 |
| 否则，中心像素非背景 | 中心像素色号 |
| 否则 | `null`（空白） |

### 设计意图

传统多数投票中，一个 14×14 的块若含 2 个黑色描边像素和 12 个粉色填充像素，会输出粉色，黑色描边就此消失。

描边优先策略的核心是：**只要描边像素占比达到阈值，就输出描边色**。阈值随「精细度」参数动态调整——精细度越高，阈值越低，越容易保留细线。

中心像素兜底则防止边缘块因票数不足被误判为空白。

## 7. 外侧轮廓补全

降采样之后，常见问题是：**最外圈的填充色格直接贴着背景，缺少黑色描边**。

### 外侧空白检测

对网格中所有 `null` 格做 BFS 洪水填充：

- 从网格四边的 `null` 格出发
- 只穿过 `null` 格扩展
- 被访问到的 `null` 格标记为「外侧空白」

未被标记的 `null` 格为「内侧空白」（如眼睛内部），不参与轮廓补全。

### 补全规则

对每个 `fill` 角色的有色格，检查四邻域：

- 若邻格是外侧空白，或邻格在网格外
- 则将该格改为图中出现最多的描边色（默认 H7 黑色）

### 效果

```
补全前：  null  F21  F21  null       补全后：  null  H7  F21  null
                ↑ 粉色贴背景                    ↑ 补为描边
```

眼睛内部的 `null` 不触发补全，因为那些 `null` 不是外侧空白。

---

## 8. 填充色合并

### 算法

对网格中每个未访问的 `fill` 格启动 BFS，扩展条件：

1. 邻格也是 `fill` 角色
2. 邻格色号与起始色的 RGB 距离 ≤ 相似度阈值

得到一个连通区域后：

- 统计区域内各色号出现次数，取众数作为目标色
- 若区域格数 < `minRegionSize`，改为邻域中出现最多的填充色

### 与旧版的关键区别

| | 旧版 | 现版 |
|---|------|------|
| 合并范围 | 所有颜色 | 仅 `fill` |
| 描边色 | 可能被合并为填充色 | 完全不参与 BFS |
| 小区域处理 | 可能置为 `null` | 合并为邻域填充色 |

---

## 9. 孤立杂点清理

最后一步，仅移除**完全孤立**的填充色单格：

- 四邻域没有任何有色格
- 该格角色为 `fill`

描边格即使只有一格、贴在背景旁，也**不会被删除**——这正是旧版「单格噪点清理」导致描边消失的根因，现已彻底移除该逻辑。

---

## 10. 参数说明

| 参数 | 范围 | 作用 |
|------|------|------|
| `detail` 精细度 | 0–100 | 控制描边保留阈值 \(r_{\text{outline}}\)，越高越容易保留细描边 |
| `bgTolerance` 背景容差 | 0–100 | 背景洪水填充的 RGB 距离容差 |
| `mergeThreshold` 相似度阈值 | 0–100 | 填充色合并的 RGB 距离容差 |
| `minRegionSize` 最小区域 | 1–10 | 小于此格数的填充色区域合并为邻色 |
| `removeBackground` | 开关 | 是否执行背景移除 |

默认值见 `lib/types/grid.ts` 中的 `DEFAULT_IMAGE_PARAMS`。

## 12. 演进与踩坑记录

本算法历经三轮迭代，以下是关键教训：

### 第一版：加权主导色 + 孤立块清理

- 按透明度与饱和度加权统计块内主导色
- 激进清除与主体断开的小色块

**问题：** 抗锯齿灰边仍残留；孤立块清理误删星星等独立细节。

### 第二版：多数投票 + 抗锯齿剥离

- 先量化再多数投票
- 从背景向内剥离低饱和度边缘像素
- 清除贴边单格噪点

**问题：** 边缘剥离在浅色主体上向内侵蚀，打出大洞；贴边噪点清理删除描边单格；右半侧主体被整块清空。

### 第三版（当前）：描边优先 + 角色分离

- 颜色三分法（背景 / 描边 / 填充）
- 描边优先降采样保留细线
- 外侧轮廓补全修复贴边缺失
- 所有合并/清理只作用于填充色

**效果：** 以 Kirby 测试图验证，贴边且无描边邻居的填充格数为 0，轮廓完整闭合。

## 13. 复杂度分析

设源图像素数为 \(N\)，网格格数为 \(M = W \times H\)，色号库大小为 \(K\)。

| 步骤 | 时间复杂度 | 说明 |
|------|-----------|------|
| 背景移除 | \(O(N)\) | BFS 每个像素最多入队一次 |
| 全图量化 | \(O(N \cdot K)\) | 最近色搜索，有 RGB 缓存 |
| 降采样 | \(O(N)\) | 每像素仅访问一次 |
| 轮廓补全 | \(O(M)\) | 两次 BFS + 一次遍历 |
| 填充合并 | \(O(M)\) | BFS 每个格最多访问一次 |
| 杂点清理 | \(O(M)\) | 单次遍历 |

对 1000×1000 源图、50×50 网格、291 色色号库，浏览器中处理耗时通常在 50–200ms。

## 14. 已知局限与后续方向

1. **色号匹配仅用 RGB 欧氏距离**，未采用 CIELAB 感知色差，某些相近色可能匹配不准
2. **描边/填充分类基于亮度启发式**，对深色系填充（如深棕轮廓 + 棕色填充）可能误判
3. **不支持抖动（Dithering）**，渐变区域会呈现色阶台阶
4. **轮廓补全仅处理外侧**，源图中本就没有描边的图案不会自动添加内描边

可能的改进：

- 引入 CIELAB 色差匹配
- 支持用户手动指定描边色
- 可选 Floyd-Steinberg 抖动模式
- 超分辨率预处理（2× 量化后再降采样）
