Long Blog Post

2019年5月29日 · 阅读需 3 分钟
Endilie Yacop Sucipto
Maintainer of Docusaurus

这是一篇关于如何优化长文博客阅读体验的指南。无论你是写技术文档还是个人随笔，好的排版和交互细节都能让读者更愿意读完全文。

阅读体验的核心：让读者始终知道「我在哪」以及「还剩多少」。

排版与可读性

长文阅读的首要问题是视觉疲劳。通过合理的行高、字号和段落间距，可以显著降低读者的认知负担。

字体与行高

中文正文建议行高在 1.75–1.85 之间，字号不小于 16px。系统字体栈（PingFang SC、Microsoft YaHei 等）在各类设备上都能保证清晰渲染。

段落结构

每段控制在 3–5 行以内，避免大段文字堆砌。适当使用加粗突出关键概念，用列表拆分步骤型内容。

导航辅助

超过 1000 字的文章，读者很容易迷失方向。以下功能可以大幅改善导航体验：

  • 目录（TOC）：根据标题自动生成，点击即可跳转
  • 阅读进度条：顶部细线实时显示阅读进度
  • 回到顶部：滚动一定距离后出现，方便快速返回
  • 侧边栏固定：最近文章列表随页面滚动保持可见

主题与无障碍

支持浅色/暗黑模式切换，能照顾不同光线环境下的阅读需求。切换偏好后写入 localStorage，下次访问自动恢复。

减少动画

对于开启了「减少动态效果」的用户，应通过 prefers-reduced-motion 媒体查询关闭非必要的过渡动画，避免造成不适。

性能建议

静态站点部署在 GitHub Pages 上时，注意以下几点：

  1. 压缩 CSS/JS 资源，启用浏览器缓存
  2. 图片使用 WebP 格式并设置合理尺寸
  3. 避免在首屏加载过多第三方脚本
  4. 使用 <link rel="preload"> 预加载关键资源

总结

优化长文阅读体验并不需要复杂的技术栈——清晰的排版、贴心的导航辅助、以及尊重用户偏好的主题切换，就能让博客从「能读」变成「好读」。希望这篇指南对你有所帮助。
